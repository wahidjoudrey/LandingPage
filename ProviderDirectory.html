Da Vinci PDex Plan-Net Provider Directory

An Overview of the Provider Directory API using HL7 FHIR Resources

What is PDex Plan-Net?

The Da Vinci PDex Plan-Net Implementation Guide defines a FHIR-based standard for provider directory data exchange between payers, third parties, and consumers. It helps meet the CMS Interoperability and Patient Access Rule by enabling a standardized, queryable provider directory.

It supports the publication of provider network data including practitioners, their roles, locations, services, and the organizations they represent.

Core FHIR Resources in Plan-Net

Organization

Represents a healthcare organization (payer, hospital, group practice). It may be a top-level network or a sub-organization within a larger network.

Key Fields: resourceType,meta, id, language, active, name, type, telecom, address, contact

API Usage Examples

All resources are accessible via standard FHIR RESTful APIs. Examples:

  • GET /Location?_id=123 – Retrieve a specific location
  • GET /Practitioner?name=Smith – Search for practitioners by name
  • GET /PractitionerRole?specialty=cardiology – Find roles by specialty
  • GET /Organization?identifier=123456 – Lookup organization by ID
  • GET /HealthcareService?type=urgent-care – Find specific service types
